Rue's Rube Goldberg machine: Dog Dairy Generator

A Rube Goldberg machine consist of a series of simple unrelated devices; the action of each triggers the initiation of the next, eventually resulting in achieving a stated goal.
In this project, I made my own Rube Goldberg machine in the form of "A Dog Dairy Generator": simply enter your dog's breed and your zip code, it will present you the dairy of your dog at that day.

Here's my initial diagram:

I started with the Dog Api. The first obstacle I encountered was to use a textbox rather than a button. So that whatever breed the user enters, the DogApi will return an image of that breed.

Next, I added the Weather API from OpenWeather, which allow people to enter a zipcode and get a string that describes the weather condition of that region.

Then, my machine branchs out into two directions depending on the current weather condition: if the weather isn't rainy, it will return a "yes" gif from YesOrNo Api.
Otherwise, it will return a "no" gif. I used an If statement to achieve this function.

In the "if it's not rainy" branch, I initially wanted to bring up a Map ApI that finds the nearest dog park based on the input address. I tried to use GoogleMap APi at first but failed to get a valid authentication.
Then, I decided switch to an OpenMap APi but encountered lots of difficulties even with in-class troubleshooting. So I gave up this idea, and uses a Cocktail Api instead.

On the "if it's rainy branch, I simply brought in a DIY Api that returns fun indoor activities for spending time with my dog at home"

Lastly, I cleaned up and organized my code, deleting the extra submit buttons so that the users only have to submit once after filling both textbox.

And here's what I got: a dog dairy generator!
